# 全站性能优化与品牌体验升级

## 背景
- 用户确认按照前一轮诊断结论推进站点提速与外观设计升级。
- 本次目标是在不引入构建流程的前提下，完成一轮可直接上线的静态站点优化。

## 阶段计划

### 阶段一：资源加载优化
- [x] 移除正式页面未使用的第三方脚本与样式
- [x] 用原生交互替换 `jQuery + Bootstrap JS + Slick + Magnific Popup + Counterup`
- [x] 移除非联系页地图请求，并将联系页地图改为更轻量的展示方式
- [x] 收敛图标依赖，移除 Font Awesome

### 阶段二：品牌视觉升级
- [x] 升级首页首屏与核心信息区，让页面更像高品质纸品目录册
- [x] 升级产品页卡片、规格区与询盘区的质感表达
- [x] 升级服务页流程与能力呈现，强化专业、可靠的供应链叙事
- [x] 调整关于页与联系页的细节质感，使全站视觉统一

### 阶段三：文档与验证
- [x] 更新前端文档中的资源加载与视觉约束
- [x] 执行 JS 语法检查
- [x] 执行翻译 JSON 与键一致性检查
- [x] 复核任务文档 TODO 并归档

## 关键约束
- 继续保持纯静态站点形态，不引入包管理级依赖到仓库中
- 代码注释与文档统一使用中文
- 优先复用现有结构与翻译键，避免无必要扩大多语言维护面
