# 静态站点治理

## 背景
- 站点主业务内容已经成型，但 `service.html`、`pricing.html` 仍保留明显模板残留。
- 页面头部、页脚和联系方式通过复制粘贴分散在多个 HTML 中，维护成本偏高。
- 多个页面的 `meta` 信息与 `html lang` 语义未统一，影响品牌一致性与 SEO 基础质量。

## 目标
- 将 `service.html` 清理为可正式对外展示的业务页面，并将 `pricing.html` 收口为不参与索引的遗留跳转页。
- 统一主站页面的标题、描述与语言语义。
- 收敛共享页头页脚，移除误导性模板链接与假功能区块。

## 实施阶段
1. 收敛共享结构与信息架构
2. 改造 `service.html` 并移除 `pricing.html` 曝光
3. 统一页面语义信息与翻译
4. 本地校验与文档回写
