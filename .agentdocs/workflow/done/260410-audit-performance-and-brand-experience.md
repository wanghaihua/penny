# 站点性能与品牌体验诊断

## 背景
- 用户希望从整个项目出发，分析可以提升访问速度的方案，并给出更符合纸业供应商“专业、绿色、可靠”定位的外观与动效方向。
- 当前站点为纯静态多语言官网，核心页面包括首页、关于页、服务页、产品页和联系页。

## 诊断范围
- 页面资源加载结构
- 图片、图标、第三方依赖与多语言机制
- 当前视觉表达、品牌质感与动效策略

## 现状发现

### 性能与加载
- 5 个正式业务页面当前都加载相同的 10 个 CSS 文件引用与 13 个脚本引用，基础本地 CSS 体积约 312 KB，本地 JS 体积约 300 KB，尚未计入 Google Maps 远程脚本与图标字体实际下载体积。
- `index.html`、`about.html`、`service.html`、`project.html` 都加载了 Google Maps API 与 `plugins/google-map/map.js`，但页面本身没有 `#map` 容器，属于纯冗余请求。
- `service.html`、`contact.html` 也加载了 Magnific Popup、Slick、Counterup 等插件，但页面并未实际使用对应能力，说明目前资源未按页面职责拆分。
- 图标体系同时依赖 Font Awesome 与 Themify，CSS 文件之外还会触发图标字体请求；仅字体源文件总量已超过 1 MB，适合改为按需 SVG 图标。
- 产品页直接使用原始 PNG 作为卡片图与弹窗大图，单页去重后本地资源约 1.49 MB，是当前最重的正式页面。
- 首页主视觉、内页头图和 CTA 仍依赖 CSS 背景图与固定背景，缺少响应式图片策略，浏览器无法像 `<img>` / `<picture>` 那样更细粒度地控制加载。
- `js/site-layout.js` 与 `js/lang-switch.js` 中存在语言识别、翻译拍平等重复逻辑，虽然不是首要性能瓶颈，但会增加维护和初始化复杂度。

### 视觉与品牌表达
- 当前站点已经具备“绿色、克制、商业化”的基础方向，但整体仍带有较明显的模板继承痕迹，缺少纸业供应商独有的材质感与供应链可信度表达。
- 首屏、内页头图、卡片与表格之间的品牌叙事还不够连贯，更多是在“模块美化”，还没有形成“原纸能力 -> 转换制造 -> 品控交付 -> 可持续承诺”的完整视觉路径。
- 动效目前以滚动显现为主，节奏比较稳，但还没有把“纤维、层压、工艺、交付节点”这些行业特征转化为更有记忆点的微动效语言。

## 优化建议

### 第一阶段：先解决最影响访问速度的问题
- 按页面按需加载插件与样式，移除非联系页的 Google Maps 相关脚本，首页/服务页/联系页停止加载未使用的 Slick、Magnific Popup、Counterup。
- 产品页图片改为 WebP/AVIF，并拆分“列表缩略图”和“放大查看大图”两套资源；列表首屏图保留较高质量，其余缩略图懒加载。
- 将首页与内页头图从单一 CSS 背景图改为可响应式控制的图片方案，至少补充移动端版本。
- 收敛图标方案，优先保留一套图标体系，核心高频图标改为内联 SVG。

### 第二阶段：进一步提升首屏和稳定性
- 逐页拆分为基础公共包与页面专属包，例如 `base.css/base.js`、`home.css`、`products.css`、`contact.css`。
- 精简模板遗留 CSS，减少仍保留在 `css/style.css` 中但已不再需要的旧样式、渐变和背景规则。
- 为关键图片补充明确尺寸或比例容器，降低布局抖动风险。
- 若继续使用嵌入式翻译 + JSON 刷新策略，可将共享语言工具抽离为单一模块，避免重复初始化。

### 第三阶段：部署层提速
- 为静态资源开启长期缓存策略，并让文件名带版本号或查询参数，以便更新时安全失效。
- 若当前仍直接依赖基础静态托管，可考虑接入 Cloudflare 一类 CDN 做边缘缓存、压缩与图片格式协商。
- 对图片、CSS、JS 使用 Brotli/Gzip 压缩，并检查是否已经启用 HTTP/2 或 HTTP/3。

## 品牌视觉方向建议

### 视觉主题
- 主基调建议从“绿色模板站”升级为“纸张质感 + 供应链可信度”的工业轻奢风格：纸张米白作底、森林绿作结构、铜金或温暖卡其作极小面积强调。
- 背景不需要花哨渐变，更适合使用低对比纸纹、压纹、纤维纹理或网点式浅层图案，让页面更像高品质样册而不是通用企业模板。
- 图片选择上应减少泛泛的办公或团队图，增加卷纸母卷、压花纹理、包装细节、码垛装柜、质检场景等真实供应链素材。

### 首页结构建议
- 首屏改为“采购视角”表达：左侧一句清晰价值主张，右侧不是普通装饰图，而是产品层纹、包装细节或生产场景的高质感拼接视觉。
- 将 `18+ Years / 60+ Markets / 15-25 Days / OEM` 一类数字升级为“能力证据条”，加入更强的标签层级、分隔线和轻微材质底板。
- 在首页补充“Why global buyers trust us”或“Quality & compliance checkpoints”模块，把专业和可靠从口号变成流程证据。

### 内页设计建议
- 产品页可改为“目录册”风格：分类卡 + 关键规格摘要 + 应用场景标签 + 询盘入口，减少单纯图片墙的模板感。
- 服务页适合做成“订单旅程”界面，把询盘、打样、确认、生产、检验、装柜、文件交付串成带编号的时间轴。
- 联系页可以弱化地图存在感，把重点放到联系渠道、响应时效、支持语言、常见询盘信息模板上。

## 动效建议
- 首屏可加入极轻的分层入场：标题、按钮、能力标签分 60-90ms 错峰出现，避免一次性全部弹出。
- 卡片悬停不建议只是上浮，可以加入“纸张翻页感”的轻微阴影变化和描边强化，位移保持在 4-6px 内。
- 产品页图片悬停可使用缓慢缩放 + 轻微遮罩显现，模拟样册翻阅，而不是强烈放大。
- 服务流程模块适合做滚动时间轴激活效果：当前步骤高亮，前后步骤弱化，强化订单推进感。
- 可持续理念区域可使用非常轻的纤维流线、圆点扩散或叶脉式线框装饰，但必须遵守 `prefers-reduced-motion`。

## 建议优先级
- P0：移除无用第三方脚本、按页拆插件、优化产品页图片
- P1：拆分 CSS/JS 公共包与页面包，收敛图标方案
- P2：重做首页首屏与供应链信任表达，统一各页面叙事
- P3：补充更成体系的品牌纹理、流程动效与部署层 CDN/缓存优化
