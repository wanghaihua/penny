# 首屏共享布局提速

## 背景
- 当前正式页面的页眉和页脚依赖 `js/site-layout.js` 在 `DOMContentLoaded` 后注入，因此首屏会先出现空挂载点。
- `js/lang-switch.js` 当前在 HTTP 场景下优先拉取远程翻译 JSON，且使用 `cache: 'no-store'`，会进一步延后页眉导航与其他 `data-i18n` 文案的可见时间。
- 页面底部还先加载了 jQuery、Slick、Counterup 和全站地图脚本，再执行共享布局脚本，导致页眉页脚感知上更慢。

## 目标
- 让页眉和页脚在 HTML 解析过程中就能渲染出来，而不是等整页脚本跑完。
- 让多语言文案优先基于嵌入式翻译立即显示，再后台刷新远程 JSON。
- 保持共享布局单一维护入口，不回退到每个页面手写整套页眉页脚。
